数据结构与算法系列

2020-06-23 4 min read

套路：

  1. 先想个暴力解法
  2. 再进行优化，常见的算法思路：索引、指针、递归、分治，贪心、动态规划等
  3. 看能否套用现成的数据结构：堆、栈、队列、树、图等。
  4. 时间与空间的交换，比如哈希表。
  5. 对输入进行预处理，比如排序。
  6. 在瓶颈处分析，比如O(nlogn)+O(n^2)，就要想办法优化O(n^2)。

极端条件的判断：
数组为空、字符串为空、数量为0、空指针。

空间复杂度
注意递归调用也是要花费空间的，因为要保存调用栈。递归调用n次，则空间复杂度为O(n)。

时间复杂度
O(n):循环C*n次，其中C为常数
O(n^2):循环C *n^2次，其中C为常数

链表

1 基本定义

package com.lsz.数据结构与算法.linkedlist;

public class LinkedList<E> {

    private class Node {
        private E e;
        private Node next;

        public Node(E e) {
            this(e, null);
        }

        public Node(E e, Node next) {
            this.e = e;
            this.next = next;
        }

        public Node() {
            this(null, null);
        }

        @Override
        public String toString() {
            return e.toString();
        }
    }

    private Node head;
    private int size;

    public LinkedList() {
        head = null;
        size = 0;
    }

    //获取元素个数
    public int getSize() {
        return size;
    }

    //链表是否为空
    public boolean isEmpty() {
        return size == 0;
    }

    //在链表头添加节点,对用户来讲，传入的是E，提供的接口隐藏了内部实现Node
    public void addFirst(E e) {
//        Node node = new Node(e);
//        node.next = head;
//        head = node;
        //上面三句话可用下面这一句代替
        head = new Node(e, head);
        size++;
    }

    //在链表的index(从0开始)处添加新的元素
    //fixme 链表中通常不用索引，此处为练习
    public void add(int index, E e) {
        if (index < 0 || index > size) {
            throw new IllegalArgumentException("wrong index!");
        }

        if (index == 0) {
            addFirst(e);
        } else {
            //开始遍历找到index-1处的元素
            Node prev = head;
            for (int i = 0; i < index - 1; i++) {
                prev = prev.next;
            }
            //找到了index-1处的元素
//            Node node = new Node(e);
//            node.next = prev.next;
//            prev.next = node;
            //上面三行可用这一行代替
            prev.next = new Node(e, prev.next);
            size++;
        }
    }

    //在链表末尾添加元素
    public void addLast(E e) {
        add(size, e);
    }
}

2 dummyHead

基本定义中的添加元素的add方法需要对头节点的情况做特殊处理，很不优雅，可以引入dummyHead的概念，这样一来，所有的节点都拥有前置节点，所以就无需对头节点做特殊处理。

3 递归的删除链表元素

递归分三步：
明确函数功能：要清楚你写这个函数是想要做什么；
确认递归终点：递归一定要有递归终点，不然会永远递归下去，禁止套娃；
找出递推关系：开始实现递归，一步一步递推出最终结果。
明白了这个道理，代码也就不难了，如下：

/** 作用就是保证以head为头节点的链表中不存在值为val的节点
     * @param head 链表头节点
     * @param val  要删除的值
     * @return 删除值后，剩下的链表的头节点
     */
    public Node removeElement(Node head, E val) {
        //如何递归调用
        //1.确定递归终点，当head为null时
        if (head == null) return null;
        //2.假设[head->next, ...]这区间内已经删除了val
        Node node = removeElement(head.next, val);
        //3.分情况讨论
        if (head.e == val){
            //head需要删除
            return node;
        }else {
            //head不需要删除
            head.next = node;
            return head;
        }
    }
